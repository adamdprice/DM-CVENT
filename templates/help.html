User Guide

Complete reference for the Dealmakers Cvent → HubSpot integration.

Overview

This portal connects Cvent (your event registration platform) with HubSpot (your CRM). When an attendee registers for a Dealmakers event in Cvent, this integration lets you push their details, registration data, and order information directly into HubSpot — creating and updating Contacts, Attendee records, and Deals automatically.

ℹ️
Nothing is ever pushed to Cvent. This is a one-way sync: data flows from Cvent into HubSpot only.

How the integration works

Each sync is triggered manually (per attendee, or batch "Sync All") or on a scheduled basis. When triggered, the integration:

  1. Pulls the attendee's full profile and order history from the Cvent API
  2. Finds or creates a matching Contact in HubSpot (matched by email address)
  3. Finds or creates an Attendee custom object record (matched by Cvent attendee ID)
  4. Sets all attendee properties from Cvent data
  5. Creates associations between the attendee and the relevant HubSpot Events, Festivals, and Sponsors
  6. Creates or updates Deals to capture revenue

Transaction steps

Attendees in Cvent can have multiple order transactions (e.g. initial registration, then a paid upgrade). The sync processes each transaction as a separate step — this is what you see as numbered tabs in the Training report. Phantom transactions (identical to a previous step) are automatically skipped to prevent duplicate deals.

Skip rules

The following attendees are skipped or handled differently:

  • No email address — sync fails with an error (email is required to find/create a Contact)
  • Not "Accepted" status and never previously synced — skipped entirely (no HubSpot writes)
  • Cancelled and previously synced — only the cvent_reg_status and cvent_cancelled fields are updated on the Attendee; no new associations or deals are created

Deal currency

Select the currency for all deals created for this event from your HubSpot portal's configured currencies, then click Save. This sets the deal_currency_code property on every deal created during sync.

Registration types & paths

Read-only badges showing the registration types (e.g. Paying Delegate, Speaker) and paths (e.g. Public, Invite Only) configured in Cvent for this event. These drive the attendee labels and deal scenarios during sync — no action required here.

Admission items

Cvent admission items represent what an attendee paid for (e.g. "Paying Delegate Pass"). Each must be linked to one or more HubSpot Event objects so the sync knows which event to associate the attendee and deal to.

Click + to open a picker, select the relevant HubSpot event(s), and save. The mapping is stored directly on the HubSpot Event record's cvent_admission_item_ids property.

⚠️
If admission items are not mapped, the sync cannot determine which HubSpot event to link the attendee to, and no deal will be created. Always map admission items before running a live sync.

Quantity items

Cvent quantity items are add-on purchases (e.g. workshop fees, gala dinner tickets). You can optionally map each quantity item to a HubSpot Product — when mapped, the deal created for that quantity item will be associated with that product in HubSpot.

Quantity item mappings are stored locally in your browser (not the server). They apply when you sync manually or use "Sync All" from the browser, but not during scheduled auto-syncs.

Registration questions

Cvent registration questions (e.g. "Job title", "How did you hear about us?") can be mapped to HubSpot Attendee properties. The question's answer is then written to that property on every sync. See Question mappings for full details.

Question mappings

Cvent lets attendees answer custom questions during registration (e.g. "Job title", "How did you hear about us?", "Dietary requirements"). Question mappings let you route those answers into specific HubSpot Attendee properties automatically on each sync.

How to set up a mapping

  1. From the home screen, click Configure question mappings →, or from an event's setup page scroll to Registration questions
  2. Select the event from the dropdown
  3. Find the question you want to map and click +
  4. Either select an existing HubSpot Attendee property, or create a new one on the spot
  5. Save — the mapping is stored server-side and applies immediately on the next sync
💡
Mappings are per-event, keyed by Cvent question ID. You can create new HubSpot Attendee properties directly from the mapping UI — they are added to the Registration Questions property group.

Training vs Live mode

🧪 Training mode
  • No writes to HubSpot
  • Shows a full preview report
  • Displays which Contact and Attendee would be found or created
  • Shows the full deal plan with amounts
  • Shows event, festival, and sponsor associations
  • Safe to run as many times as needed
✅ Live mode
  • Creates/updates records in HubSpot
  • Processes each transaction step in order
  • Creates Contact if not found
  • Creates or updates Attendee record
  • Creates associations to Events, Festivals, Sponsors
  • Creates or updates Deals with amounts and tax
✅
Always run Training mode first to verify the deal amounts, event associations, and property values before committing a live sync.

Sync All

From the Attendees list, click Sync All Attendees to sync every attendee in the event in one go. This always runs in Live mode — a confirmation dialog warns you before starting.

  • Attendees are processed sequentially, one at a time
  • A progress bar and counter show how many have been processed
  • Click Cancel to stop after the current attendee finishes
  • A summary at the end shows how many succeeded, were skipped, or had errors
⚠️
Sync All can take several minutes for large events. Do not close the browser tab while it is running.

Scheduled (auto) sync

Enable the Auto-sync toggle on an event to have it synced automatically every 30 minutes in the background, without any manual action.

How it works

  1. The portal's background worker checks for due events every 2 minutes
  2. An event is "due" if auto-sync is enabled and it hasn't run in the last 30 minutes
  3. The worker fetches all Cvent attendees for the event
  4. Attendees who haven't changed in Cvent since their last successful sync are skipped (incremental — only new/modified records are processed)
  5. Each due attendee is synced in live mode
ℹ️
Scheduled syncs require a PostgreSQL database (DATABASE_URL environment variable) to persist state. Without it, the toggle will appear to work but the schedule won't survive a server restart.
💡
Quantity item → HubSpot Product mappings (configured in Event Setup) only apply to manual/Sync All runs from the browser. Scheduled syncs do not apply these mappings.

Records created in HubSpot

A single live sync for one attendee can create or update up to 4 types of records in HubSpot:

👤 Contact Found by email, or created if new. First name, last name, email set on creation.
🏷️ Attendee Custom object (2-44005420). Found by Cvent ID, or created. All properties updated every sync.
💰 Deal One per event (admission). Additional deals for quantity items. Created if conditions met.
📦 Product link Quantity item deals are linked to a HubSpot Product if a mapping exists.

Association labels (Attendee → Event)

The association between an Attendee and a HubSpot Event carries a label that describes the attendee's role at that event:

  • Paying Delegate — paid admission attendee
  • Speaker - Non Sponsor — speaker not associated with a sponsor
  • Speaker - Sponsor — speaker who is also sponsor-linked
  • Dealmakers Guest — guest/complimentary attendee
  • Sponsor Executive — executive from a sponsoring organisation
  • Sponsor Client — client brought by a sponsor

Deal creation scenarios

The sync engine analyses each attendee's registration type, order history, and number of events to determine how deals should be structured. Here are the main scenarios:

Standard (single event)

Attendee registered for one event, paid admission.

→ 1 deal created for the event, amount = admission total

Standard (multi-event)

Attendee registered for two events, single lump-sum payment.

→ 1 deal per event; admission amount split 50/50

Paying Delegate upgrade

Existing attendee, 2 events, 2+ transactions (e.g. initial free + paid upgrade).

→ Upserts both existing deals; splits cumulative total 50/50

Speaker upgrade

Speaker type, 2 events, paid additional transaction for a second event.

→ 1 new deal for the event they are Paying Delegate at; speaker labels applied to both events

Sponsor upgrade

Sponsor Executive/Client, 2 events, paid upgrade for admission at one event.

→ 1 deal for the event where they are Paying Delegate (not the sponsor event)

Quantity items

Attendee purchased add-ons (workshop, gala dinner, etc.).

→ 1 separate deal per quantity item, net amount only; linked to product if mapped
⚠️
No deal is created when: the attendee's status is not Accepted, or the order amount is zero or negative. Free registrations (speakers, guests) do not generate deals unless they later pay.
